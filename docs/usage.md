# Руководство пользователя

## Таймер

Блок таймера появляется на странице судьи сразу после нажатия **Старт →**.

| Кнопка | Действие |
|---------|----------|
| **250 с** | Запустить обратный отсчёт с 4:10 |
| **50 с** | Запустить обратный отсчёт с 0:50 |
| **■ Стоп** | Остановить и сбросить таймер |

- Нажатие  250 с или 50 с пока идёт другой — **перезапускает** таймер.
- При истечении — 00:00 красным.
- На **scoreboard** таймер отображается автоматически синхронно с запуском судьи.
  Обновление страницы зрителей не нужно.

---

## Перед каждым соревнованием

1. Открыть ссылку судьи (сохранена в телефоне/мессенджере)
2. Нажать **+ Новое соревнование**
3. В модале выбрать таб — **«Пары»** или **«Индивидуальный финал»**
   (по умолчанию открывается «Индивидуальный финал»)
4. Ввести фамилии 8 стрелков, Enter — следующая строка
5. Нажать **Старт →**
6. Убедиться что scoreboard открыт на экране для зрителей

> Это может сделать любой у кого есть ссылка судьи — не только владелец аккаунта.

---

## Во время соревнования (судья)

1. Открыть сохранённую ссылку судьи
2. После каждой серии — ввести сумму серии в ячейку (цифра видна на экране установки)
3. Enter — следующая строка, Tab — следующий столбец
4. Всё сохраняется автоматически
5. Рейтинг внизу страницы обновляется сразу — видно кто вылетает

### Навигация в таблице

| Клавиша | Действие |
|---------|---------|
| Enter | ↓ следующая строка (тот же столбец) |
| Tab | → следующий столбец |
| Shift+Tab | ← предыдущий столбец |
| ↑ ↓ | строка выше/ниже |
| ← → | столбец влево/вправо |

## Таблица ввода (страница судьи)

Вид таблицы зависит от типа соревнования, выбранного при создании.

**Пары:**
- Колонки: Щит | Фамилия | **5 | 5 | 5 | 3 | 3 | 3** | Σ
- 8 строк, разделители между парами (щит 1+2, 3+4, ...)
- Заголовок колонки рейтинга — «Пара»

**Индивидуальный финал:**
- Колонки: Щит | Фамилия | **5 | 5 | 2 | 2 | 2 | 2 | 2 | 2 | 2** | Σ
- 8 строк подряд, без разделителей
- Заголовок колонки рейтинга — «Имя»

- Заголовки колонок = количество выстрелов в серии/раунде
- Автосохранение при уходе из ячейки
- Сортировка не отображается в таблице ввода — судья видит стрелков в
  порядке щитов; актуальная сортировка по очкам — в рейтинге внизу страницы

---

## Индивидуальный финал (8 стрелков)

Формат: 2 квалификационные серии по 5 выстрелов (`s1`, `s2`), затем 7 раундов
по 2 выстрела (`s3`..`s9`) — итого 24 выстрела на человека. Судья вводит
результат **суммой за серию/раунд**, а не по одному выстрелу.

### Выбывание не реализовано в коде

Программа **не отслеживает и не подсказывает**, кто выбывает. Рейтинг внизу
страницы — это просто сортировка по общей сумме очков, как и в режиме пар.
Судья определяет выбывание глазами, по рейтингу, и после этого просто не
вводит очки выбывшему стрелку в следующих колонках — его ячейки остаются
пустыми, а сумма, зафиксированная на момент выбывания, продолжает
отображаться в рейтинге (просто больше не растёт).

### ⚠️ Правило отсечки — как определить, кто выбывает

1. После **квалификации** (`s1`, `s2` — серии 5+5) отсечки **не происходит
   вообще** — все 8 стрелков остаются в игре. Это чисто квалификационный этап.
2. Начиная с **первого раунда по 2 выстрела** (колонка `s3`), после каждого
   раунда выбывает **один** стрелок.
3. **Важно:** после первого выбывания рейтинг целиком уже не подсказывает,
   кто следующий на выход — выбывший стрелок с «замороженной» суммой обычно
   и так висит внизу списка. Смотреть на «последнего во всём рейтинге» —
   **ошибка**, это часто ранее выбывший, а не корректный кандидат.
4. **Правильное правило:** выбывает стрелок с наименьшей суммой **среди тех,
   кому судья только что ввёл очки в текущем раунде** (то есть среди
   стрелков с заполненной самой правой на данный момент колонкой) — а не
   среди всех 8 подряд.
5. При равенстве очков между кандидатами на выбывание — решение остаётся за
   судьёй (в реальности — перестрелка); программа это не обрабатывает.

Если не следовать пункту 1 — есть риск, что судья по ошибке применит правило
«наименьший среди заполнивших колонку» уже после `s1`/`s2` и объявит
выбывшего раньше срока, хотя все 8 стрелков после квалификации ещё в игре.

## Страница зрителей (scoreboard)

- Чёрный фон, белый текст, лидер выделен жёлтым
- Таблица: `#` | Пара | Итого
- В колонке «Пара» рядом с именем каждого участника — его индивидуальная сумма в скобках:
  `Иванов (24.0) / Петров (18.0)` (скобки появляются после первой введённой серии)
- Обновляется мгновенно при каждом введённом значении
- При старте нового соревнования страница перезагружается автоматически
- **Авто-реконнект:** при обрыве сети подключение восстанавливается
  автоматически, данные подтягиваются — вручную перезагружать не нужно
  (индикатор в углу: `● live` / `● connecting`)
- **Полноэкранный режим:** двойной клик по странице или клавиша **F**

## Смена токена судьи

Если сменился судья или нужно отозвать доступ:

**Supabase → SQL Editor** → открыть готовый файл `sql/rotate-token.sql`,
заменить `НОВЫЙ_ТОКЕН` на новую строку → **Run**.

Либо вручную:

```sql
CREATE OR REPLACE FUNCTION get_judge_token()
RETURNS text LANGUAGE sql SECURITY DEFINER STABLE AS $$
  SELECT 'НОВЫЙ_ТОКЕН';
$$;
```

Отправить судье новую ссылку:
```
https://sst1xx.github.io/mushketon-final/judge/?token=НОВЫЙ_ТОКЕН
```

Код репозитория при этом **не меняется**.
