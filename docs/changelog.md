# История изменений и идеи

## Что сделано

- Настроен GitHub репозиторий и GitHub Pages
- Настроен Supabase: таблицы, RLS, Realtime, GRANT права
- Авторизация через токен в URL (без email, без формы входа)
- Scoreboard с Realtime подпиской (WebSocket, без polling)
- Страница судьи: модал участников, таблица ввода, рейтинг внизу
- Логика пар: щит 1+2, 3+4 (не два стрелка на одном щите)
- Фиксированный формат: 4 пары / 8 стрелков (щиты 1–8)
- Заголовки серий: 5 5 5 3 3 3 (количество выстрелов в серии)
- Enter → переход вниз по строкам; Tab → следующий столбец
- Рейтинг на странице судьи без дублирования данных
- В строке рейтинга (judge и scoreboard) рядом с именем каждого участника
  показывается его индивидуальная сумма в скобках (например, «Иванов (24.0) / Петров (18.0)»);
  если серии ещё не введены — скобки не показываются

## Упрощение эксплуатации

- Единый `config.js` в корне: ключи Supabase в одном месте (были в двух HTML)
- `supabase_setup.sql` делает всё сразу: таблицы, RLS, GRANT, Realtime (убраны ручные шаги)
- все SQL-скрипты собраны в папке `sql/`: `supabase_setup.sql`, `add-timer-fields.sql`, `rotate-token.sql`
- `sql/rotate-token.sql` — готовый сниппет смены токена
- Документация приведена к текущей модели (`shooters`, токен в URL, 4 пары / 8 человек)
- scoreboard: авто-реконнект при обрыве связи + полноэкранный режим

## Таймер

- Судья запускает обратный отсчёт (250 с или 50 с) на своей странице
- Страница зрителей показывает тот же таймер синхронно через Realtime
- Состояние хранится в БД — при перезагрузке страницы зрителей таймер подхватывает актуальное состояние
